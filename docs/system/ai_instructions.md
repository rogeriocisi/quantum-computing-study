# AI Development Guidelines

This document establishes the rules and standards for all code and documentation generated by the AI coding assistant within this repository.

## 1. Specification First
**Rule**: Before implementing any new complex algorithm or module, a Specification (Spec) must be created and stored in `docs/system/specs/`.
- The Spec serves as a contract defining requirements, success criteria, and technical constraints.
- Once the Spec is established, the implementation must strictly follow it.

## 2. Documentation Coupling
**Rule**: Every new algorithm, utility, or module created must be accompanied by its respective technical documentation.
- **Location**: All technical docs must be placed under `docs/reference/<category>/`.
- **Content**: Documentation must include:
    - Academic or technical references.
    - Overview and logical flow.
    - API Reference (parameters and return types).
    - Usage examples.

## 2. Language Standard
**Rule**: All source code comments, docstrings, and documentation files must be written in **English**.
- This ensures international accessibility and follows professional development standards.

## 3. Quality Assurance
**Rule**: All new code must include a corresponding test suite in the `tests/` directory.
- Tests should cover structural integrity, mathematical correctness (via simulation), and reversibility (where applicable).
